## 2. `resume.md` Generation

### 2.1 Input Directory

Generator reads from:

```text
$workspace/memory/claude/
$workspace/memory/codex/
$workspace/memory/shared/
```

It must not read:

```text
$workspace/memory/templates/
$workspace/memory/examples/
$workspace/memory/invalid/
$workspace/memory/MEMORY.md
$workspace/memory/resume.md
```

Rationale: entry files are source of truth. `MEMORY.md` is a non-authoritative index and must not feed generated resume output.

### 2.2 Selection Criteria

Include an entry only when all of these are true:

```text
schema_version: 1
status: active
visibility: shared
priority: critical or high
```

Also require:

```text
type in decision, gotcha, convention, verification, handoff, note
id present
title present or recoverable from id
frontmatter closed by a second ---
```

Skip entries when:

```text
frontmatter is missing or malformed
schema_version is not 1
status is missing or not active
visibility is not shared
priority is normal or low
type is invalid
id is missing
```

Warn, but do not fail, when:

```text
title is missing: use id in generated resume
created/updated missing: omit timestamp from item
tags malformed: omit tags
body cannot be read: include metadata only
```

### 2.3 Ordering

Sort included entries in this deterministic order:

1. `priority: critical`
2. `priority: high`
3. Type order:
   ```text
   gotcha
   convention
   decision
   handoff
   verification
   note
   ```
4. Newest `updated` first, fallback to `created`, fallback to filename sort descending.
5. Filename ascending as final tie-breaker.

Rationale:

- Critical operational hazards should appear first.
- Conventions should appear before decisions when they affect immediate startup behavior.
- Handoffs matter more than historical verification.

### 2.4 Length Budget

Hard budget:

```text
MAX_RESUME_CHARS=8000
MAX_ENTRY_CHARS=900
MAX_ENTRIES=12
```

Recommended output target:

```text
4000 to 6500 chars
```

Truncation rules:

1. Never truncate the header, generation metadata, warnings, or "How to use" section.
2. Include at most 12 selected entries.
3. For each entry, prefer:
   - title
   - type/priority/path
   - `## Summary` paragraph if present
   - otherwise first body paragraph after H1
4. If one entry exceeds `MAX_ENTRY_CHARS`, truncate the summary and append:
   ```text
   [truncated]
   ```
5. Stop adding entries before exceeding `MAX_RESUME_CHARS`.

If there are too many selected entries, add a warning:

```text
Warning: resume truncated at 12 entries or 8000 chars. Scan memory/ directly for full context.
```

### 2.5 Generated Format

`memory/resume.md` should be fully overwritten with this generated template:

```markdown

1. Read this file at startup when using abg-open --resume.
2. Treat it as a hint list, not authority.
3. When correctness matters, open the linked entry file.
4. Ignore skipped/corrupt entries unless debugging memory itself.

## Selected Entries

### 1. Do not schedule current-minute one-shot CronCreate

- id: 20260614-022100-codex-gotcha-no-current-minute-cron
- type: gotcha
- priority: critical
- updated: 2026-06-14T02:21:00+08:00
- path: memory/examples/good-gotcha.md
- tags: agentbridge, cron, polling

Summary:
One-shot CronCreate uses minute-level cron. A target less than 120 seconds away can be missed or scheduled far in the future.

## Skipped Entries

- memory/examples/bad-broken-frontmatter.md: malformed frontmatter

If no entries are selected, this section says:

No active shared critical/high priority memory entries were found.
```

In real generation, examples must not be selected because `memory/examples/` is excluded.

### 2.6 Atomic Write

Write with temp file plus atomic rename:

```zsh
tmp="$memory_dir/.resume.md.tmp.$$"
printf '%s\n' "$content" > "$tmp"
mv "$tmp" "$memory_dir/resume.md"
```

If write fails:

- remove temp file if present
- print warning to stderr
- return non-zero to caller
- caller must continue launching

### 3.4 zsh Function Skeleton

Add a zsh function before the launch flow, after argument parsing helpers or before `target=""`.

This is intentionally dependency-free and conservative. It uses shell scanning, not a full YAML parser.

```zsh
generate_memory_resume() {
  local workspace="$1"
  local memory_dir="$workspace/memory"
  local resume_file="$memory_dir/resume.md"
  local tmp="$memory_dir/.resume.md.tmp.$$"
  local now
  now="$(date '+%Y-%m-%dT%H:%M:%S%z')"

  if [[ ! -d "$memory_dir" ]]; then
    echo "[abg-open] Memory resume skipped: memory directory not found: $memory_dir" >&2
    return 0
  fi

  if [[ ! -w "$memory_dir" ]]; then
    echo "[abg-open] Memory resume warning: memory directory not writable: $memory_dir" >&2
    return 1
  fi

  # Implementation detail:
  # - scan only memory/claude, memory/codex, memory/shared
  # - exclude examples, templates, invalid, MEMORY.md, resume.md
  # - parse frontmatter until second ---
  # - accept only schema_version=1, status=active, visibility=shared, priority=critical/high
  # - collect records into arrays or temp lines
  # - sort by priority/type/timestamp/path
  # - write bounded Markdown to $tmp, then mv $tmp $resume_file
}
```

The final implementation can use helper functions:

```zsh
memory_field() {
  # $1 file, $2 key
  # print the first simple scalar value for key from frontmatter
}

memory_summary() {
  # $1 file
  # print first paragraph under ## Summary, fallback to first body paragraph
}

memory_type_rank() {
  case "$1" in
    gotcha) echo 1 ;;
    convention) echo 2 ;;
    decision) echo 3 ;;
    handoff) echo 4 ;;
    verification) echo 5 ;;
    note) echo 6 ;;
    *) echo 9 ;;
  esac
}
```

### 3.6 Concrete Call Site

Add:

```zsh
if (( resume )); then
  if [[ "${ABG_MEMORY_RESUME:-1}" == "0" ]]; then
    echo "[abg-open] Memory resume disabled by ABG_MEMORY_RESUME=0"
  else
    echo "[abg-open] Refreshing memory resume: $workspace/memory/resume.md"
    if ! generate_memory_resume "$workspace"; then
      echo "[abg-open] Memory resume warning: generation failed; continuing launch" >&2
    fi
  fi
fi
```

Important: this block must not run for normal `abg-open` without `--resume`.

### 3.7 Error Handling

| Condition | Behavior |
|---|---|
| `memory/` missing | print skipped message, continue launch |
| `memory/resume.md` missing | create it if `memory/` writable |
| entry unreadable | skip entry, record in "Skipped Entries", continue |
| corrupt frontmatter | skip entry, record reason |
| no selected entries | write valid resume with empty-state message |
| temp write fails | warn, continue launch |
| `mv` fails | warn, remove temp if possible, continue launch |
| sort/scanning bug | catch through function non-zero, warn, continue launch |

Because the script has `set -euo pipefail`, the call must be inside `if ! generate_memory_resume ...; then` so a non-zero return does not abort the launcher.

Within the function, avoid command substitutions that can fail under `set -e` without `|| true`.

### 3.8 Must Not Do

Do not:

- prepend resume content to Claude prompt
- send resume content through AgentBridge
- patch `CLAUDE.md` dynamically
- patch `AGENTS.md` dynamically
- modify `.agentbridge/config.json`
- rewrite `MEMORY.md`
- move invalid entries automatically
- require `jq`, `yq`, Python, Node, Ruby, or network access

## 9. Open Design Decisions

Decision: Should normal priority shared entries be included?

Answer for Phase 1: no. Keep selection limited to `critical` and `high` to control resume length and avoid startup noise.

Decision: Should Claude/Codex private visibility entries be included for their own side?

Answer for Phase 1: no. `resume.md` is shared startup context. Private per-agent resume can be a later phase.

Decision: Should `MEMORY.md` be regenerated too?

Answer for Phase 1: no. That is a separate indexing feature and increases concurrency risk.

Decision: Should `abg-open` prompt agents to read `resume.md`?

Answer for Phase 1: no dynamic prompt injection. Static protocol in `CLAUDE.md` and `AGENTS.md` is enough.

## 10. Acceptance Summary

Phase 1 is accepted when:

- `abg-open --resume` refreshes `memory/resume.md` before opening sessions.

- no dynamic patching, prompt injection, external dependency, or `MEMORY.md` rewrite is introduced.
